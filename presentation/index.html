08 / XSS Detection

XSSScanner — Nonce-Based Reflection Probing

Algorithm:

  1. สร้าง random 8-char hex nonce ต่อ URL
  2. Inject payload ใน query parameters
  3. ค้นหา nonce กลับมาใน response body
  4. พบ nonce → Reflected XSS Confirmed

Payload Templates (8 รูปแบบ):

<script>/*NONCE*/</script>
"><img src=x onerror=/*NONCE*/>
"><svg onload=/*NONCE*/>
<input onfocus=/*NONCE*/ autofocus>
... + 4 more contexts
CWE-79 A05:2025-Injection
09 / SQLi Detection

SQLiScanner — 3-Phase Detection

1
Error-Based

Inject ', '', ' OR ''=' → ตรวจหา SQL error regex ใน response

~3 requests · เร็วที่สุด
2
Boolean-Based Blind

เปรียบเทียบ body size ระหว่าง 1=1 และ 1=2 (diff > 10%)

~4 requests · ปานกลาง
3
Time-Based Blind

SLEEP(3), pg_sleep(3), WAITFOR DELAY → response time ≥ baseline + 2000ms

~40s max · ช้าที่สุด
CWE-89 A05:2025-Injection MySQL · PostgreSQL · MSSQL

12 / Auth & BruteForce

Authentication + Brute Force

🔑 Form-Based Login

  • Auto-detect username/password fields จาก HTML
  • ดึง hidden fields (CSRF tokens) อัตโนมัติ
  • Session cookies ส่งต่อทุก request
  • ใช้ x/net/html parse form

💀 Brute Force Login

  • ทดสอบ username × password combinations
  • 3-signal success detection:
    • Failure keywords ใน body
    • URL redirect path เปลี่ยน
    • Session cookie ปรากฏ
  • Safety cap: max 500 attempts, min 100ms delay
  • รายงานเป็น Critical A07 · CWE-307

14 / Risk Scoring

Risk Scoring — OWASP-Inspired Formula

สูตรคำนวณ RiskScore (per finding type):

RiskScore =
( IncidenceRate% × 0.30 )
+ ( ExploitScore × 10.0 )
+ ( ImpactScore × 20.0 )
+ ( OccurrenceRatio × 10.0 )
IncidenceRate% — สัดส่วน URL ที่มีช่องโหว่นี้ (เทียบกับ URL ทั้งหมด)
ExploitScore — ความง่ายในการโจมตี (CVSS exploitability, 0–10)
ImpactScore — ความรุนแรงของผลกระทบ (CVSS impact, 0–10)
OccurrenceRatio — จำนวนครั้งที่พบ / URL ทั้งหมด
Impact weighted ×20 vs Exploit ×10 — สะท้อน OWASP methodology ที่ให้ความสำคัญกับผลกระทบมากกว่าความง่าย

ค่าคะแนนตาม Severity (ที่ Incidence 100%):

SeverityExploitImpactMax Score
Critical 9.09.8 326
High 7.58.0 275
Medium 5.06.5 220
Low 3.04.0 150
ตัวอย่าง: Missing HSTS (High) พบ 10/10 URL
( 100% × 0.30 ) + ( 7.5 × 10 ) + ( 8.0 × 20 ) + ( 1.0 × 10 )
= 30 + 75 + 160 + 10 = 275.00

Limitations & Future Work
